No generic rounded cards, pill soup, gradient blobs, or floating SaaS panels. Geometry should come from the product metaphor and workflow.

Make Floop feel designed at the component level.

The next move is not just a better layout. It is a new component vocabulary: tile edges, lane rulers, waterlines, notched decisions, ledgers, and depth controls that make the interface recognizable before any copy is read.

Anti-slop principles

These rules keep the Floop direction from falling back into common generated UI shapes.

Design system stance
No generic cards

Use tiles and ledgers

Rectangular, aligned, grid-native surfaces with strong edges and purposeful internal structure.

No pill soup

Use stamped state

Status is a small rectangular stamp tied to the lane, not a decorative capsule floating on every item.

No soft blobs

Use waterlines

Motion and brand should come from horizontal/vertical flow, not abstract organic shapes.

No fake depth

Use actual depth

Summary, evidence, and logs should change layout density and geometry, not only color.

Adoption rules

These are the practical constraints I would use before touching production CSS.

Implementation guardrails

Replace

  • Rounded ticket cards with lane-striped tile tickets.
  • Pill tabs with hard-edged segmented depth controls.
  • Modal detail-first flow with a notched desktop inspector.
  • Timeline feed styling with evidence ledger rows.
  • Generic side nav items with waterline navigation groups.

Keep disciplined

  • Use radius only for native controls where the platform expects it, and keep it at 0-4px.
  • Make every decorative line carry structure: lane, depth, ledger, deck, or selection.
  • Prefer dividers, insets, and measured spacing over shadows.
  • Let status color be stamped and sparse.
  • Keep metaphor labels paired with precise workflow labels.
